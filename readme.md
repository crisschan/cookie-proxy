## 1. 它是什么

Cookie Proxy 由三部分组成，协同工作：

```
Claude Code (Skill)
    ↓  发出请求
proxy-server（本地后台程序，端口 7070）
    ↓  通过 WebSocket 转发
Chrome 扩展
    ↓  用你的真实 Cookie 访问
目标网站（GitHub / Notion / 公司内网 / 任意登录网站）
```

**Cookie 始终留在浏览器内，永远不会传到外部。** Claude 拿到的只是网页响应内容。

## 2. 安装

### 2.1 安装 Chrome 扩展（唯一需要手动做的事）

1. 打开 Chrome，地址栏输入 `chrome://extensions/`
2. 将extension.crx拖入，安装成功

> **注意**：扩展只需安装一次。proxy-server 二进制程序会在你第一次使用时由 Claude Code 自动下载，无需手动操作。

### 2.2 确认 Python 依赖已安装

```bash
pip install httpx
```

## 3. 第一次使用

**完整流程（只有第一次需要经历）：**

```
你          →  在 Claude Code 里提问，例如：
               "帮我看看我 GitHub 上最新的 PR 列表"

Claude Code →  自动下载并启动 proxy-server（后台静默完成）

浏览器      →  弹出权限确认窗口：
               ┌─────────────────────────────┐
               │  🔐 Cookie Proxy            │
               │  Claude Code 想借用你的浏览  │
               │  器登录状态访问：            │
               │                             │
               │  github.com                 │
               │  /pulls                     │
               │                             │
               │ [允许此域名] [仅允许一次] [拒绝] │
               └─────────────────────────────┘

你          →  点击「允许此域名」

Claude Code →  返回结果
```

### 权限确认按钮说明

| 按钮 | 效果 |
|---|---|
| **允许此域名** | 该域名加入白名单，以后访问此域名不再询问 |
| **仅允许一次** | 本次放行，下次仍会询问 |
| **拒绝** | 本次请求被取消，Claude 收到拒绝消息 |

> 确认窗口 **60 秒内无操作** 自动视为拒绝。

## 4. 日常使用

第一次授权后，**直接在 Claude Code 里自然语言提问即可**：

```
"帮我整理一下 Notion 里标题包含'周报'的页面"
"从公司内网下载最新的 API 文档"
"检查我的 GitHub 有没有未处理的 issue"
"用我的账号登录状态，获取这个页面的内容：https://..."
```

Claude 会自动：
1. 检测 proxy-server 是否运行，未运行则启动
2. 通过扩展借用你的登录 Cookie 发起请求
3. 将结果呈现给你

**已在白名单的域名全程无弹窗。**

## 5. 交互操作（点击 / 填写 / 提交）

除了读取页面内容，Cookie Proxy 还支持在**已登录的页面上执行操作**：

```
"帮我在 JIRA 上把 PROJ-123 的状态改为 Done"
"在公司 OA 系统里提交今天的日报"
"帮我在内网论坛发一条回复"
```

Claude 会：
1. 找到（或打开）目标网站的 tab
2. 在页面上依次执行点击、填写、提交等操作
3. 返回操作完成后的页面标题和地址

**支持的操作类型：**

| 操作 | 说明 |
|---|---|
| 点击元素 | 点击按钮、链接、复选框等 |
| 填写输入框 | 向文本框、搜索框输入内容 |
| 提交表单 | 提交登录表单、发布表单等 |
| 等待 | 等待页面加载或动画完成 |
| 快照 | 获取当前页面的标题和 URL |

> **注意**：交互操作需要目标页面处于**已登录状态**。首次访问某域名时仍会弹出权限确认窗口，确认后 Claude 才会在该页面执行操作。

## 6. 权限管理

点击 Chrome 工具栏的 🔐 图标，打开 Popup 面板：

```
┌─────────────────────────────────┐
│  🔐 Cookie Proxy     ● 已连接   │
├─────────────────────────────────┤
│  已信任的域名                    │
│                                 │
│  github.com          [删除]     │
│  notion.so           [删除]     │
└─────────────────────────────────┘
```

- **删除** 某个域名后，下次访问该域名将重新弹出确认窗口
- 状态指示点：绿色 = proxy-server 已连接；红色 = 未连接

## 7. 扩展图标说明

| 图标状态 | 含义 |
|---|---|
| 彩色 🔐 | proxy-server 正在运行，扩展已连接，可正常使用 |
| 灰色 🔐 | proxy-server 未运行（Claude Code 发起请求时会自动启动）|

## 8. 常见问题

### Q: Claude 提示「Cannot connect to Cookie Proxy server」

**原因**：proxy-server 未启动，或 Chrome 没有打开。

**解决**：确保 Chrome 浏览器是打开状态，然后重新向 Claude 提问，它会自动启动 proxy-server。

---

### Q: Claude 提示「extension is not connected」

**原因**：扩展虽然安装了，但还未连接上 proxy-server。

**解决**：
1. 检查 `chrome://extensions/` 页面，确认 Cookie Proxy 扩展是启用状态
2. 关闭并重新打开 Chrome
3. 重新向 Claude 提问

---

### Q: 弹出了确认窗口，但我来不及点，窗口消失了

**原因**：确认窗口有 60 秒超时，超时后自动关闭并视为拒绝。

**解决**：重新向 Claude 提问，新的确认窗口会再次弹出。

---

### Q: Claude 提示「Access denied」

**原因**：你在确认窗口点击了「拒绝」。

**解决**：重新向 Claude 提问，在弹出的确认窗口中点击「允许」。

---

### Q: 我想撤销对某个网站的信任

点击 Chrome 工具栏的 🔐 图标，在已信任域名列表中点击对应域名的「删除」按钮即可。

---

### Q: proxy-server 会一直在后台运行吗？

proxy-server 是一个轻量本地进程（端口 7070，仅监听本机）。关闭 Chrome 不会停止它。如需手动停止：

```bash
# 查找进程
lsof -i :7070
# 停止进程
kill <PID>
```

下次 Claude Code 需要时会自动重新启动。

---

### Q: 首次使用时下载 proxy-server 失败

**原因**：网络问题或 GitHub Releases 地址不可达。

**解决**：
1. 检查网络连接
2. 也可以手动编译：进入 `cookie-proxy/proxy-server/` 目录，运行 `make build`，然后把生成的 `proxy-server` 二进制文件复制到 `~/.cookie-proxy/proxy-server`

---

### Q: Claude 提示「Action failed: element not found」

**原因**：交互操作的目标元素在页面上找不到，可能是页面结构变化或还未加载完成。

**解决**：
1. 确认目标页面已完全加载
2. 检查 CSS 选择器是否正确
3. 尝试在操作前加一个「等待」步骤

---

### Q: 交互操作完成了，但页面没有跳转到预期结果

**原因**：操作执行后页面可能需要时间响应（如表单提交后的跳转）。

**解决**：在关键操作后加入适当的等待时间，再用「快照」确认最终状态。

---

## 9. 安全机制

| 机制 | 说明 |
|---|---|
| Cookie 不外泄 | Cookie 只在浏览器扩展内使用，从不传出 |
| 私有地址拦截 | 禁止访问 `10.x`、`192.168.x`、`127.x` 等内网地址，防止 SSRF 攻击 |
| 仅监听本机 | proxy-server 只绑定 `127.0.0.1:7070`，局域网其他设备无法访问 |
| 响应大小限制 | 单次响应超过 10 MB 会被截断并报错 |
| 用户明确授权 | 每个新域名首次访问都需要你手动点击允许 |
| 无日志记录 | proxy-server 不记录请求内容，保护你的隐私 |

---

## 10. 卸载

1. **移除 Chrome 扩展**：`chrome://extensions/` → Cookie Proxy → 移除
2. **删除 proxy-server 数据**：
   ```bash
   rm -rf ~/.cookie-proxy
   ```
3. **删除项目文件**（可选）：删除整个 `cookie-proxy/` 目录
